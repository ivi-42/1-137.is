Let’s dig into the math a bit more. “Hardness” is a family of ideas about comparisons between problems based on reusability of algorithmic solutions. Loosely speaking, a problem is hard with respect to a class of problems if an algorithm solving can be easily transformed into an algorithm solving any problem in . You have to say what kinds of transformations are allowed, and the transformation can be different for different target problems in , but that’s the basic idea.In the Super Mario example, if you want to solve logical formulas, you can transform a hypothetically perfect mario-level-playing algorithm into a logic solver by encoding the formula as a level and running the mario-level-playing algorithm on it as a black box. Add an if statement to the end to translate “level can/can’t be finished” to “formula can/can’t be satisfied,” and the transformation is complete. It’s important for NP-hardness that the transformation only takes polynomial time. Other kinds of hardness might admit more or restrict to fewer resources. And so this is what makes Mario NP-hard, because boolean logic satisfiability is NP-hard. Any problem in NP can be solved by a boolean logic solver, and hence also by a mario-level-player. The fact that boolean logic solving is NP-hard is a difficult theorem to prove. But if we assume it’s true, you can compose the transformations to get from any NP problem to Super Mario. Sudoku is another example of a problem that can be related to computational complexity. It has been shown that solving a Sudoku puzzle is an NP-complete problem. This means that it is at least as hard as the hardest problems in the class NP, and if a polynomial-time algorithm for solving Sudoku were found, it would imply that P = NP. However, just like with the Super Mario example, the general hardness of Sudoku doesn’t necessarily translate to the difficulty of solving individual puzzles. Sudoku puzzles come in various sizes and difficulty levels. The most common Sudoku puzzles have a 9x9 grid and are designed to be solvable by humans using logical deduction. There are efficient algorithms and heuristics that can solve these puzzles quickly, as well as more complex ones. In practice, the difficulty of Sudoku puzzles for humans and the complexity of solving them algorithmically are not directly correlated. Humans are generally good at pattern recognition and can apply logical reasoning to solve moderately difficult Sudoku puzzles, whereas computers can solve even the most complex Sudoku puzzles efficiently through brute-force search or other algorithmic techniques. So, while Sudoku is an NP-complete problem in its general form, this complexity does not prevent humans or algorithms from solving real-world instances of the problem. This demonstrates that the theoretical complexity of a problem does not always dictate its difficulty in practice, and that heuristic or specialized approaches can be effective for solving instances of hard problems.
